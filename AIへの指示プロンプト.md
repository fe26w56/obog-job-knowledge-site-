# UI制作AI向けプロンプト集

## 🚀 **初回プロンプト（全体理解）**

```
あなたはUI/UXデザイナーです。学生団体OBOG向けの就活ナレッジ共有サイトのUI設計を依頼します。

【プロジェクト概要】
- 学生団体の現役生とOBOGが就活情報を安全に共有するクローズドプラットフォーム
- 承認制ユーザー登録、段階的な個人情報開示機能
- Next.js + Tailwind CSS での実装予定

【ターゲットユーザー】
1. 現役会員（大学3-4年生）：就活情報収集、主にスマホ利用
2. OB/OG会員（卒業後3年以内）：体験談投稿、PC・スマホ両方
3. 管理者：ユーザー・投稿管理、主にPC利用

【技術制約】
- Next.js 14 + Tailwind CSS
- レスポンシブ対応（Mobile First）
- アクセシビリティ対応（WCAG 2.1 AA）

添付の「UI設計指示書.md」を読んで、このプロジェクトの要件を理解してください。
理解できたら、主要な画面とユーザーフローを整理して教えてください。
```

## 📱 **画面別詳細プロンプト**

### **ダッシュボード設計**

```
【ダッシュボード画面の設計依頼】

以下の仕様でダッシュボード画面を設計してください：

■ 画面構成
- ヘッダー（ロゴ、ユーザーメニュー、通知アイコン）
- 検索バー（オートコンプリート機能）
- フィルタ（業界、企業、卒年、タグ）
- 投稿一覧（カード形式）
- フッターナビゲーション（モバイル）

■ 投稿カードの内容
- 投稿者アバター・名前・所属企業
- 投稿タイトル・概要（2行まで）
- タグ（最大3個表示）
- エンゲージメント（いいね、コメント、閲覧数）
- ブックマークボタン

■ デザイン要件
- カラー：Primary #2563EB, Secondary #10B981
- フォント：Noto Sans JP + Inter
- カード：shadow-sm, 角丸12px
- モバイルファースト設計

■ 状態別表示
- ローディング：スケルトンスクリーン
- 検索結果なし：空状態のイラスト
- 権限による表示制御（現役会員は連絡先非表示）

WireframeとVisual Designの両方を作成してください。
```

### **投稿詳細画面設計**

```
【投稿詳細画面の設計依頼】

■ 画面構成（モバイル）
- ナビゲーション（戻るボタン、タイトル、メニュー）
- 投稿者情報カード
- 投稿本文（マークダウン対応）
- タグ一覧
- アクションボタン（いいね、ブックマーク、シェア）
- コメント欄
- コメント入力フォーム

■ 投稿者情報カードの表示制御
- OB/OGには連絡先ボタンを表示
- 現役会員には「連絡先を見るには許可が必要」メッセージ
- 許可済みの場合は連絡先（メール、SNS）を表示

■ コメント機能
- ネスト対応（返信機能）
- 投稿者・コメント日時・内容
- 承認待ちの場合のステータス表示

■ インタラクション
- いいねボタンのアニメーション
- コメント開閉の折りたたみ
- 画像の拡大表示（ライトボックス）

デスクトップ版とモバイル版の両方を設計してください。
```

### **認証画面設計**

```
【認証フロー画面の設計依頼】

以下3つの画面を設計してください：

■ 1. ログイン画面
- メールアドレス入力
- OTP送信ボタン
- 学生団体のロゴ・サービス名
- 新規登録への導線

■ 2. 新規登録画面（招待URL経由）
- 基本情報フォーム（名前、大学、学部、卒業年、就職先）
- アバター画像アップロード
- OTP認証
- 利用規約・プライバシーポリシー同意

■ 3. OTP確認画面
- 6桁コード入力フィールド
- 再送信ボタン（カウントダウン付き）
- 戻るボタン

■ デザイン要件
- 信頼感のあるデザイン（セキュリティ重視）
- フォームバリデーション表示
- エラー・成功状態の表示
- プログレスインジケーター

■ アクセシビリティ
- キーボードナビゲーション
- フォーカス表示
- エラーメッセージの適切な配置

セキュリティとユーザビリティを両立したデザインでお願いします。
```

## 🎨 **管理画面設計**

```
【管理画面の設計依頼】

■ 管理ダッシュボード
- 統計カード（ユーザー数、投稿数、承認待ち数）
- 最近のアクティビティ
- クイックアクション（ユーザー承認、URL発行）

■ ユーザー管理画面
- ユーザー一覧テーブル
- フィルタ・検索機能
- 一括操作（承認、拒否、CSV出力）
- ユーザー詳細モーダル

■ 投稿管理画面
- 投稿一覧（モデレーション待ち優先表示）
- 投稿プレビューと承認・削除ボタン
- 報告された投稿の管理

■ デザイン要件
- PC利用前提（タブレット対応）
- データテーブルの見やすさ重視
- 効率的なワークフロー設計
- 権限に応じた機能表示

管理者が効率的に運営できるUIでお願いします。
```
